subgraph_id: DeploymentHashapi_version: Versiondata_source_name: Stringdata_source_address: Vec<u8>data_source_network: Stringdata_source_context: Arc<Option<DataSourceContext>>causality_region: String

Some data sources have indeterminism or different notions of time. These need to be each be stored separately to separate causality between them, and merge the results later. Right now, this is just the ethereum networks but will be expanded for ipfs and the availability chain.
